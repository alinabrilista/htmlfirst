$ git init
Создание нового репозитория
Команда создаст новый репозиторий по текущему пути. Может быть использована для инициализации git репозитория в уже существующем проекте.
Создание репозитория в существующей директории
Если у вас уже есть проект в директории, которая не находится под версионным контролем Git, то для начала нужно перейти в эту директорию. Если вы не делали этого раньше, то для разных операционных систем это выглядит по-разному:

  • для Linux: $ cd /home/user/my_project
  • для macOS: $ cd /Users/user/my_project
  • для Windows:$ cd C:/Users/user/my_project
а затем выполните команду: $ git init
Эта команда создаёт в текущей директории новую поддиректорию с именем .git, содержащую все необходимые файлы репозитория — структуру Git-репозитория.

Дополнительная информация по сылке: git-scm.com

$ git init (directory)
Создание нового репозитория по указанному пути (в указанной папке)
Команда создаст новый репозиторий в указанной папке.

$ git add
Добавление файлов для отслеживания в git
Команда добавляет файл(ы) в область staged:

  • $ git add file.txt
    Добавить только file.txt
  • $ git add folder
    Добавить папку folder и все содержимое папки
  • $ git add –A
    С флагом –A добавит все файлы в область staged
  • $ git add .
    Более сокращенный аналог команды выше, добавит все файлы по текущему пути (.)
$ git commit –am “Comment” -> Команда git commit с флагом –a также позволяет добавить все отслеживаемыефайлы в область staged

$ git commit
У каждого коммита есть свое описание(комментарий). Комментарий описывает, какие изменения в проекте вошли в коммит(что было добавлено, реализовано, удалено и т.д.) Для добавления комментария используется флаг –m
$ git commit –m “Fixed bug with subscription form”
Комментарий должен быть абсолютно у каждого коммита.

$ git branch
Работа с ветками
  • $ git branch
    Посмотреть список всех веток в репозитории
  • $ git branch (branch-name)
    Команда создаст новую ветку с указанным именем
  • $ git branch –d (branch-name)
    Удалить ветку
  • $ git branch –m (branch-name)
    Переименовать текущую ветку на указанное имя

Дополнительная информация по сылке: git-scm.com

$ git checkout
  • $ git checkout
    Перейти на ветку с указанным именем
  • $ git checkout –b (branch-name)
    Создать ветку с указанным именем и сразу перейти на неё

$ git merge

$ git merge Команда объединяет указанную ветку в текущую.
Например, если мы находимся на ветке master, то команда $ git merge hot-fix выполнит слияние ветки hot-fix в ветку master
В ветке мастер будут применены все изменения из ветки hot-fix, а ветка hot-fix останется не тронутой. Если ветка hot-fix больше не нужна, то её можно удалить.
($ git branch –d hot-fix)

$ git fetch

$ git fetch [remote-server-name]
Команда связывается с указанным удалённым репозиториеми забирает все изменения, которых ещё нет, но не сливает их с текущими наработками в локальном репозитории.
Появляется список удаленных веток, которые можно объединить с локальными в подходящий момент.
Используется, когда нужно получить все новые удаленные изменения, но не объединять их с текущими наработками.

$ git pull

$ git pull [remote-server-name] [remote-branch-name]
Команда делает, то же, что и git fetch, только автоматически сливает изменения из удаленной ветки в текущую ветку.
Подходить для случаев когда вам нужно применить все удаленные изменения в вашу текущую работу.
Эта команда последовательно выполняет:
$ git fetch -> $ git merge.

$ git push

$ git push [remote-server-name] [remote-branch-name]
Отправляет локальные изменения в удаленный репозиторий.
Нужно обязательно сделать коммит перед тем, как делать пуш. Так как локальные коммиты отправляются на удаленный сервер.
